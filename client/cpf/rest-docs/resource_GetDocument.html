GetDocument

Given the ID of a document return the the document data and optionally the document metadata if an extension is given. Without the extension just the document data will be returned in the response and the response body will have a mime type appropriate to the document data. If an extension is given (.json or .xml) then the response will be a mime multipart with the document metadata in the first part and the document data in the second part. All documents in the system have unique ids so the patient id, section id and episode id values are not required. Example URI: /document/

The following resources are applicable:

  • /document/multipart/{masterID}
  • /document/raw/{masterID}
