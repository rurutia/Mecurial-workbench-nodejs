Hi guys.

Here's a summary from the REST API meeting we just had.  If there's nothing else to add I'll create some tickets.
1
 All DTOs need javadoc documentation.
2
 All attribute names will be camel-case with a leading lower-case letter.  The Win8 code will need to be reviewed to see if the json/xml libraries used will blindly accept any changes or refactoring will be necessary.
3
 Any list of objects should be wrapped up in a list DTO rather than a straight Java array.  Otherwise the output appears as a sequence of items all with the same element name.  It is neater to have this wrapped up.
4
 Null elements should not appear in the output.
5
 SectionSummaryDTO and SectionDTO should be merged into a single SectionDTO.  Point 3 means any elements not set will not appear.
6
 Remove the response DTOs such as ResultFailDTO, ResultOkDTO, etc.  We use appropriate exceptions now.
7
 Get rid of patientComponentDTO and put its attributes directly into its subclasses.
8
 Get rid of DocTypeSpecificDTO and have a ResultMetadataDTO attribute directly in DocumentDTO.  If we're not dealing with a result then this won't appear.
9
 Change 'v=33' to 'v=321' in the RestMediaTypes.
10
 Add the response DTO to the 'vnd.infomedix' in the RestMediaType; e.g. 'vnd.infomedix.patient'.
11
 Make use of TypeHint and StatusCodes for enunciate documentation.  Especially important when returning generic response objects and the specific type cannot be inferred.
